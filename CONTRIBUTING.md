# Contributing to Jurdian Ciphers Lab

First of all, thank you for your interest in contributing to **Jurdian Ciphers Lab**.

This project is an open-source knowledge base focused on ciphers, cryptography, encoding systems, and analysis tools. Whether you're fixing a typo, improving documentation, or building a new web tool, every contribution is appreciated.

---

## How You Can Contribute

You can contribute by:

- Adding documentation for new ciphers.
- Improving existing articles.
- Fixing errors or typos.
- Adding references or practical examples.
- Creating or improving web-based tools.
- Reporting bugs.
- Suggesting new features or improvements.

---

## Before You Start

For major changes, please open an Issue first to discuss your proposal before creating a Pull Request.

Small fixes and improvements can be submitted directly.

---

## Documentation Guidelines

- Write documentation using Markdown (`.md`).
- Keep explanations clear and beginner-friendly.
- Include references whenever possible.
- Follow the existing folder structure and naming conventions.

---

## Web Tools Guidelines

When contributing a web tool:

- Use plain HTML, CSS, and JavaScript whenever possible.
- Keep the tool lightweight and easy to understand.
- Make sure it works locally in a modern browser.
- Add comments when the implementation is complex.

---

## Pull Requests

Before submitting a Pull Request:

- Test your changes.
- Keep commits focused on a single purpose.
- Write clear and descriptive commit messages.

---

## Code of Conduct

Please be respectful and constructive when interacting with other contributors.

By contributing to this project, you agree to follow the project's Code of Conduct.

---

Thank you for helping improve **Jurdian Ciphers Lab**
